[cite_start]ระบบสมาชิกและการเข้าสู่ระบบ** *(Registration & Login)* [cite: 106] | - [cite_start]สร้าง API Route `/api/auth/register` และ `/api/auth/login`<br>- เขียนระบบเข้ารหัสผ่านด้วย **bcrypt** ก่อนบันทึก [cite: 27][cite_start]<br>- เขียนระบบออก Token และตรวจสอบด้วย **JWT** [cite: 27] | - สร้างหน้าจอ Form สำหรับสมัครสมาชิกและ Login<br>- เขียนโค้ดจัดการ State เพื่อเก็บรักษาสถานะการ Login<br>- พัฒนาการแสดงผล UI ตามสถานะ Auth | - [cite_start]**Lead Architect:** ออกแบบตาราง `users` ลงในฐานข้อมูล พร้อมกำหนด Primary Key [cite: 38, 39]<br>- **Integration Engineer:** ร่างโครงสร้าง Request/Response Payload |
| **2. [cite_start]แคตตาล็อกคลาสเรียนแบบไดนามิก** *(Dynamic Catalog Display)* [cite: 104] | - [cite_start]สร้าง API Route `GET /api/classes` สำหรับส่งข้อมูลรายการคลาสเรียนทั้งหมด [cite: 25][cite_start]<br>- เขียน SQL Query ดึงข้อมูลจากฐานข้อมูลโดยใช้ Parameterized Queries [cite: 40] | - [cite_start]สร้างหน้าจอ UI สำหรับแสดงรายการคลาสเรียน<br>- เขียนฟังก์ชัน `fetch()` ดึงข้อมูลจาก API มา Render (ห้าม Hard-code ข้อมูลสินค้าเด็ดขาด) [cite: 9] | - [cite_start]**Lead Architect:** ออกแบบตาราง `classes` เก็บรายละเอียดวิชา รวมถึงคอลัมน์ `max_capacity` [cite: 89][cite_start]<br>- วาด **Sequence Diagram** จำลองการทำงานแบบ Round-trip ของ GET request [cite: 8] |
| **3. [cite_start]ระบบตัวกรองอัจฉริยะ** *(Interactive Filtering)* [cite: 105] | - พัฒนา SQL Query ให้รองรับการรับค่า Query Parameters มากรองข้อมูลบนเซิร์ฟเวอร์ | - [cite_start]สร้างแผงควบคุม UI สำหรับกรองข้อมูล (ช่องค้นหา, ราคา, หมวดหมู่) [cite: 105][cite_start]<br>- **เขียนระบบ Debouncing** ให้กับช่องค้นหาเพื่อหน่วงเวลาก่อนยิง API [cite: 14] | - [cite_start]**UX Engineer:** ตรวจสอบและทดสอบ UI Flickering (การกระตุกของหน้าจอ) เพื่อให้การโต้ตอบเป็นไปอย่างลื่นไหล [cite: 14] |
| **4. [cite_start]ตะกร้าสะสมรายการจอง** *(Add to Cart / Booking)* [cite: 107] | - (เน้นการจัดการสถานะฝั่งไคลเอนต์เป็นหลักในขั้นตอนนี้ก่อนเข้าสู่กระบวนการสั่งซื้อ) | - [cite_start]สร้างปุ่ม "กดจองที่นั่ง" ในแต่ละคลาสเรียน<br>- **ใช้เทคนิค Event Delegation** ติดตั้ง Listener ที่ Parent Container ของรายการคลาสเพื่อประหยัด Memory [cite: 12, 13][cite_start]<br>- เขียนระบบ **localStorage Continuity** เก็บข้อมูลตะกร้าไม่ให้หายเมื่อรีเฟรชหน้าเว็บ [cite: 20, 21] | - [cite_start]**UX Engineer:** พัฒนา **Hydration Logic** เพื่อดึงข้อมูลตะกร้าเดิมกลับมาเซ็ต [cite: 22][cite_start]<br>- ควบคุมสเตตัสในตะกร้าให้เป็น Single Source of Truth [cite: 19] |
| **5. [cite_start]ระบบยืนยันการจองและเช็คที่นั่ง** *(Checkout & Capacity Logic)* [cite: 108] | - [cite_start]สร้าง API Route `POST /api/bookings` สำหรับส่งข้อมูลการยืนยันจอง [cite: 33][cite_start]<br>- **The Gatekeeper Pattern:** เขียน Service คำนวณนับจำนวนยอดจองปัจจุบันเทียบกับ `max_capacity` บน Server [cite: 90]<br>- หากที่นั่งเต็มแล้ว ให้ส่ง HTTP Status Code **409 Conflict** ปฏิเสธรายการทันที [cite: 91] | - สร้างหน้าสรุปรายการจอง พร้อมปุ่มยืนยันคำสั่งซื้อ<br>- เขียนฟังก์ชันส่งคำขอแบบ POST ไปยังหลังบ้าน<br>- เขียน UI ดักรับกรณีเซิร์ฟเวอร์ส่ง Status 409 เพื่อแจ้งเตือนผู้ใช้ [cite: 91] | - **Lead Architect:** ออกแบบตาราง `bookings` (ตาราง Transaction)<br>- **Integration Engineer:** ตรวจสอบการแกะ Payload ด้วย Middleware `express.json()` [cite: 32, 33] |
| **6.
